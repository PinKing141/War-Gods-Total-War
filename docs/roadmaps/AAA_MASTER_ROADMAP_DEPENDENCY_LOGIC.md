# Master roadmap dependency logic

The roadmap order is not random.

## Early dependency chain

```text
Validation -> UI Copy -> Data Architecture -> World Scale 1
```

Why:

```text
Validation prevents broken content.
UI copy prevents debug language spreading.
Data architecture gives new content a clean home.
World Scale 1 then safely adds 16-20 serious powers.
```

## Simulation dependency chain

```text
Core AI -> Characters -> Dynasties -> Courts/Law -> Memory
```

Why:

```text
Faction AI creates state behaviour.
Characters make decisions personal.
Dynasties make decisions generational.
Courts and law create internal politics.
Memory lets past events shape future decisions.
```

## Society dependency chain

```text
Culture/Faith -> Institutions -> Diplomacy -> Economy
```

Why:

```text
Culture and faith define identity.
Institutions turn identity and wealth into power.
Diplomacy lets powers negotiate and manipulate.
Economy gives them material reasons to fight or compromise.
```

## Conflict expansion chain

```text
Non-state actors -> Espionage -> Technology -> Disasters -> Magic -> Naval
```

Why:

```text
Non-state actors create threats outside normal countries.
Espionage lets characters and institutions scheme.
Technology changes military/economic capacity.
Disasters create external shocks.
Magic adds rare fantasy consequences.
Naval uses economy, ports, non-state piracy and diplomacy.
```

## Scale and release chain

```text
World Scale 2 -> Tools -> World Scale 3 -> Presentation -> Release
```

Why:

```text
35-50 entities test larger systemic density.
Tools are needed before 100+ entities.
100+ entities need validation, tools and balance reports.
Presentation comes after systems are worth showing.
Release comes after long-run stability.
```
