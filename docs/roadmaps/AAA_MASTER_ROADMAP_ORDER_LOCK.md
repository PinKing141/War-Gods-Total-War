# Master roadmap order lock

Do not reorder the early gates unless there is a bugfix emergency.

## Locked early order

```text
1. Baseline/freeze check
2. Data validation and technical quality
3. UI copy, SPAG and presentation language
4. Data architecture and scenario foundation
5. World Scale Gate 1: 16-20 serious powers
```

## Reason

```text
Validation must exist before large content.
UI copy must be clean before UI text spreads.
Data architecture must exist before many new files.
World expansion needs all three.
```

## Rule

If a future task tries to add a major system before these gates, treat it as out of order unless it is explicitly fixing a blocking bug.
