# Final note on the master roadmap

The roadmap deliberately puts validation, UI language and data architecture before big content expansion.

Reason:

```text
If the project expands to 20+ factions before validation and UI copy are clean, every later phase becomes harder to fix.
```

Immediate order:

```text
1. Data validation and technical quality
2. UI copy, SPAG and presentation language
3. Data architecture and scenario foundation
4. World Scale Gate 1: 16-20 serious powers
```

Then the project can safely move into:

```text
simulation depth
characters
dynasties
courts
memory
culture/faith
institutions
diplomacy
economy
non-state actors
espionage
technology
disasters
magic
naval systems
larger world scale
tools
presentation
release
```
